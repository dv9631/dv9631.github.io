what is fmea?

a systematic procedure to analyse failure modes of a system their causes and effects on the system

when is the fmea to be done?

preferabilly early in the development cycle so that removal or mitigation is cost effective . can be innitiated as soon as the system is defined enough to be presented as a functional block diagram takes place along with the design process

how to draw a functional block diagram?

refer IEC 61078- analysis for system reliability - reliability block diagram method

what is the properties of the methodology?
  • bottom up methodology
  • suitabel for idividual failues
  • not suitable for dependant failures
  • not sutable for failures depending on sequence of events

      5.2.2 system structure

      5.2.2.1 information on system structure

      provide detailed information on the system :

      • system elements with characteristics
      • performance
      • roles and functions
      • logical connection btw elements
      • redundancy level
      • nature of redundancy level
      • importance of the system within whole
      • inputs of system
      • output of system
      • changes in system for various operation modes

      5.2.2.2 defining system boundary for the analysis

      the interactions of the system with its surrounding must be defined, what are the inputs the system recives and
      what are the outputs the system gives must be defined

      if the system has multiple input outputs, functionally complex with multiple interconnections,a study boundayry can be defined
